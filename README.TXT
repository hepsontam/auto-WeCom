简介：
	本测试项目基于Android Debug Bridge（adb）来无线调控安卓手机，并以企业微信作为调试App，在语音识别后自动完成健康上报（或者略过语音识别，直接完成自动上报）。

适用环境：
	Windows + Android + 无线局域网 （如使用手机热点、家庭无线路由器等）。

使用方法：
	1、手机开启“开发者模式”，允许“USB调试”，并查询设备的局域网IP（IPv4地址）。
	   （有需要者可将手机设置为“文件传输模式”。）
	2、初次使用时，须运行“0.有线初始化.exe”，并输入设备IP，即“设备名称"，被记录于文件“DeviceName.txt”中。
	3、语音识别：运行“1.语音识别.exe”，对麦克风说出“健康上报”的命令（需获取麦克风权限），若识别结果通过，则自动进行健康上报。
	     健康上报：运行“2.健康上报.exe”，则对IP所属设备进行调控，完成上报（该程序独立于语音识别功能）。
	4、若需要增添或删除设备名称，则运行“3.修改设备名称.exe”，并按提示操作，修改原有记录。	

备注：
	1、移动设备在有线与无线调试的切换时，“USB调试”的状态可能会改变，初始化时需要多留意手机设置。
	2、“DeviceName.txt”可记录的最大设备数为9个。
	3、当连接设备失败时，程序会在项目目录下创建“ErrorDevices.txt”，并记录当前的设备名称于其中，方便后续的排查工作。
	4、为保程序能正确运行，请勿将“.exe”文件直接重命名。
